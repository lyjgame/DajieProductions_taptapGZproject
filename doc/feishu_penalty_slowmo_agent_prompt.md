# 点球 Slowmo：给 GameJam Agent 的开发提示词

## 使用方式

把下面这段提示词交给 GameJam 开发 agent，用来指导它开始制作《点球 Slowmo：54321》。如果 agent 一次不能完成全部内容，优先让它完成 MVP 可玩闭环。

## 总提示词

你要制作一款名为《点球 Slowmo：54321》的 2D 双人联机点球大战小游戏。

游戏核心是：两名玩家在线 1v1 对战，一方扮演点球手，一方扮演守门员。点球手射门后，游戏进入 5 秒 slowmo 子弹时间，屏幕显示巨大倒计时 `5、4、3、2、1`。点球手用鼠标拖拽足球决定射门方向、角度和力量；守门员用鼠标拖拽守门员决定扑救方向，并可以通过松开后双击屏幕触发提前移动。倒计时结束后，根据双方输入结算进球、扑出或射失。

主题 `54321` 必须直接参与玩法：每次点球的 slowmo 操作窗口就是 5 秒倒计时。

## 必须遵守的方向

- 做 2D 游戏，不做 3D。
- 做 Q 版角色，不做写实复杂人物。
- 使用点球手背后伪透视视角。
- 全程只用鼠标操作。
- 优先做好友房联机，不优先做随机匹配。
- 优先同步最终输入和结算结果，不做复杂实时物理同步。
- 不做完整足球比赛，只做点球大战。
- 不做复杂道具系统。
- 不做复杂角色养成。
- 不做排行榜，除非所有核心内容已完成。

## 核心玩法规则

- 游戏采用现实点球大战规则。
- 双方轮流射门和守门。
- 常规阶段每名玩家各罚 5 球。
- 5 轮结束后，进球多的一方获胜。
- 如果平局，进入突然死亡。
- 突然死亡中双方各踢 1 球，如果一方进球另一方没进，则进球方获胜。

## 画面要求

使用 2D 点球手背后伪透视视角：

- 画面下方：点球手、足球、点球点。
- 画面上方：球门、守门员、球网、观众席。
- 画面中间：足球飞行轨迹、slowmo 特效、倒计时。

美术风格：

- 2D Q 版角色。
- 大头短身。
- 动作夸张。
- 表情明显。
- 球场可简化，但球门、球网、草地和观众席要能看出来。

## 点球手操作

点球手只用鼠标操作足球：

1. 鼠标点击足球。
2. 按住并向后拖动。
3. 松开后确认射门。

输入含义：

- 点击足球的位置决定触球点。
- 拖动方向决定射门角度。
- 拖动距离决定射门力量。
- 拖得越远，力量越大。

需要显示：

- 拖拽箭头。
- 力量提示。
- 足球飞行拖尾。

## 守门员操作

守门员只用鼠标操作守门员：

1. 鼠标点击守门员。
2. 按住并向左右两侧或目标方向拖动。
3. 松开后守门员扑向该方向。
4. 松开后双击屏幕，守门员提前移动，扑救范围更大。

无输入规则：

- 如果守门员不输入，默认守中路。
- 踢向中上的球会被接住。
- 踢向中下的球会被用脚挡出。
- 踢向左右死角的球无法防住。

## 判定规则

球门分为 5 个区域：

- 左上
- 左下
- 中路
- 右上
- 右下

最小判定规则：

- 射门区域和扑救区域相同：扑出。
- 射门区域和扑救区域不同：进球。
- 守门员提前移动：可以覆盖相邻区域。
- 高力量射门：更难扑，但有射偏风险。
- 守门员无输入：自动防中路。

不要一开始做复杂真实物理。先做清楚的 2D 区域判定。

## 联机规则

先做好友房：

- 创建房间。
- 显示房间码。
- 输入房间码加入。
- 双方准备。
- 房主先当点球手。
- 每球后交换攻守。

同步方式：

- 不同步每一帧足球位置。
- 只同步点球手最终输入。
- 只同步守门员最终输入。
- 由房主或服务器统一结算。
- 把结算结果同步给双方。
- 双方使用同一个比分和结果。

## 必须做的 MVP

第一优先级必须完成：

- 标题页。
- 创建房间。
- 加入房间。
- 房间等待。
- 双方准备。
- 2D 点球场景。
- 点球手拖拽射门。
- 守门员拖拽扑救。
- 守门员双击提前移动。
- 5 秒 slowmo。
- 巨大 `5 4 3 2 1` 倒计时。
- 5 区域判定。
- 进球 / 扑出 / 射偏结果。
- 比分板。
- 双方轮流。
- 5 球 + 突然死亡。
- 最终胜负页。

做到这些以后，游戏才算可以提交。

## 有时间再做

核心玩法完成后再做：

- 裁判吹哨动画。
- 入场短镜头。
- 观众欢呼动画。
- 门将提前移动残影。
- 球网震动。
- 门柱和横梁。
- 关键球慢动作回放。
- 点球手庆祝和懊恼动画。
- 守门员庆祝和失落动画。
- 不同球衣颜色。
- 排行榜。

## UI 文案

必须包含：

- 创建房间
- 加入房间
- 房间码
- 等待对手加入
- 准备
- 你是点球手
- 你是守门员
- 拖拽足球射门
- 拖拽守门员扑救
- 双击提前移动
- 等待对手输入
- 进球！
- 扑出！
- 射偏！
- 突然死亡
- 你赢了
- 你输了

## 验收标准

提交前必须确认：

- 两名玩家可以进入同一房间。
- 能完整打一场点球大战。
- 每球都有 5 秒 slowmo 倒计时。
- 点球手能拖拽射门。
- 守门员能拖拽扑救。
- 进球和扑救结果清楚。
- 比分不会错。
- 5 球后能结束。
- 平局后能突然死亡。
- `54321` 主题在第一分钟内能被看懂。

## 最重要的提醒

先让一球能玩，再做一整局。  
先做本地逻辑，再接联机。  
先做判定，再做动画。  
先做清楚反馈，再做华丽特效。  
不要偏离 2D 点球 slowmo 这个核心。
