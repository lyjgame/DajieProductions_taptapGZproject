# 点球 Slowmo：联机房间流程与同步规则

## 目标

这份文档用于指导《点球 Slowmo：54321》的双人在线联机设计。目标是用最简单、最稳定的方式完成 1v1 房间对战，避免在 GameJam 中把网络同步做得过重。

## 联机设计原则

- 优先做好友房，不优先做随机匹配。
- 优先同步最终输入，不做复杂实时物理同步。
- 每次点球只同步必要数据：射门输入、扑救输入、倒计时状态、结算结果。
- 结果最好由同一端或服务器统一判定，避免双方结果不一致。
- 网络卡顿时宁可等待锁定，也不要让双方画面出现不同比分。

## 房间流程

### 创建房间

房主点击“创建房间”后：

- 生成房间码。
- 房主进入等待界面。
- 显示房间码和复制按钮。
- 显示“等待对手加入”。

### 加入房间

另一名玩家点击“加入房间”后：

- 输入房间码。
- 成功加入后进入准备界面。
- 如果房间不存在，提示“房间不存在”。
- 如果房间已满，提示“房间已满”。

### 双方准备

两名玩家都进入房间后：

- 显示双方昵称或玩家 1 / 玩家 2。
- 显示准备按钮。
- 双方都准备后开始比赛。

## 身份分配

推荐规则：

- 第 1 轮由房主先当点球手，加入者先当守门员。
- 每次点球结束后双方交换攻守。
- 第 1 球：玩家 A 射门，玩家 B 守门。
- 第 2 球：玩家 B 射门，玩家 A 守门。
- 依次循环，直到双方各踢 5 球。

这样规则简单，玩家容易理解。

## 比赛状态

建议把比赛拆成以下状态：

- WaitingRoom：等待房间玩家。
- ReadyCheck：双方准备。
- Intro：入场展示。
- RunUp：准备主罚。
- SlowmoInput：5 秒 slowmo 输入。
- Resolve：结算本次点球。
- SwitchRole：交换攻守。
- MatchEnd：比赛结束。

## 每球同步流程

### 1. 开始本次点球

同步内容：

- 当前轮次。
- 当前点球手。
- 当前守门员。
- 当前比分。
- 本球开始时间。

双方画面进入同一阶段。

### 2. 进入 Slowmo

点球手触球后：

- 双方同时进入 5 秒 slowmo。
- 显示倒计时 `5 -> 4 -> 3 -> 2 -> 1`。
- 点球玩家进行射门拖拽。
- 守门员玩家进行扑救拖拽。

### 3. 输入锁定

倒计时结束时：

- 点球玩家输入锁定。
- 守门员玩家输入锁定。
- 若玩家没有输入，使用默认输入。

默认输入：

- 点球玩家无输入：低力量射向中路。
- 守门员无输入：守中路。

### 4. 发送输入

点球玩家发送：

- 射门目标区域。
- 射门力量档位。
- 触球点类型。
- 是否射偏或门柱判定所需参数。

守门员玩家发送：

- 扑救目标区域。
- 是否提前移动。
- 是否无输入守中路。

### 5. 统一结算

推荐由房主或服务器统一计算结果：

- 判断射门是否射偏。
- 判断是否打门柱。
- 判断守门员是否覆盖射门区域。
- 得出结果：进球、扑出、接住、脚挡、射偏、门柱。

然后把结果同步给双方。

### 6. 播放结果

双方根据同一结果播放动画：

- 点球手动画。
- 守门员动画。
- 足球轨迹。
- 球网或扑救反馈。
- 比分变化。

### 7. 进入下一球

结算完成后：

- 更新比分。
- 交换攻守。
- 判断是否比赛结束。
- 如果未结束，进入下一次点球。

## 同步数据建议

每次点球可以同步一个简化数据包：

```text
shotIndex: 当前第几次点球
kickerId: 点球玩家
keeperId: 守门员玩家
shotZone: leftTop / leftBottom / center / rightTop / rightBottom
powerLevel: low / medium / high
contactType: top / bottom / left / right / center
keeperZone: leftTop / leftBottom / center / rightTop / rightBottom
keeperEarlyMove: true / false
keeperNoInput: true / false
result: goal / saved / caught / kickedOut / miss / post / crossbar
scoreA: 玩家A比分
scoreB: 玩家B比分
```

## 延迟处理

GameJam 版本不建议做复杂预测。推荐简单处理：

- slowmo 输入期间允许本地显示拖拽反馈。
- 倒计时结束后进入“等待对手输入”状态。
- 双方输入都收到后再结算。
- 如果短暂延迟，就显示“等待对手”。
- 如果长时间无响应，按无输入规则处理。

建议超时时间：

- slowmo 输入结束后等待 3 秒。
- 3 秒内未收到输入，使用默认输入。
- 10 秒以上无响应，提示连接异常。

## 断线处理

最小版本：

- 对手断线，显示“对手断开连接”。
- 玩家可返回房间或主菜单。

可选版本：

- 允许 10 秒重连。
- 重连成功后继续当前比分。
- 重连失败则判定当前比赛中止。

## 防止结果不一致

必须避免双方各算各的导致比分不同。

推荐做法：

- 只允许一个权威端结算。
- 另一个玩家只提交输入。
- 结算结果作为唯一结果广播。
- 双方 UI 和比分都使用广播结果。

如果没有服务器：

- 由房主作为权威端。
- 加入者发送输入给房主。
- 房主计算结果后发给加入者。

## 联机 MVP

最小可做版本：

- 创建房间。
- 输入房间码加入。
- 两人准备。
- 房主先射门。
- 双方轮流攻守。
- slowmo 后同步双方最终输入。
- 房主统一结算。
- 同步比分和轮次。
- 比赛结束显示胜负。

暂时不做：

- 随机匹配。
- 排行榜。
- 观战。
- 复杂重连。
- 实时物理同步。
- 帧级同步。

## 给 Agent 的注意事项

- 联机同步优先保证比分一致。
- 不要同步每一帧足球位置。
- 只同步输入和结算结果。
- 先实现好友房，再考虑匹配。
- 如果联机太难，先做本地双人模拟流程，保留接口方便之后接入。
