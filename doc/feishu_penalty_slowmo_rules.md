# 点球 Slowmo：判定规则与数值建议

## 目标

这份文档用于把《点球 Slowmo：54321》的射门、扑救、slowmo 和结果结算规则写清楚，方便 GameJam agent 直接实现。规则优先追求简单、公平、反馈明确。

## 核心判定原则

- 不做复杂真实物理，使用 2D 区域判定。
- 足球最终落点由点球玩家的鼠标拖拽决定。
- 守门员扑救区域由守门员玩家的鼠标拖拽决定。
- 结果由“射门区域、扑救区域、力量、提前移动、无输入中路防守”共同决定。
- 所有判定都应该能被玩家看懂，避免黑箱随机。

## 球门 5 区域

球门分为 5 个主要区域：

- 左上
- 左下
- 中路
- 右上
- 右下

2D 画面中可以把球门划成可视或隐形区域：

- 左上：球门左侧上半部分。
- 左下：球门左侧下半部分。
- 中路：球门中央区域，可以同时包含中上和中下。
- 右上：球门右侧上半部分。
- 右下：球门右侧下半部分。

## 点球玩家输入判定

点球玩家操作：

1. 点击足球。
2. 按住并向后拖动。
3. 松开后确认射门。

输入拆解：

- 点击位置：决定触球点。
- 拖动方向：决定射门目标方向。
- 拖动距离：决定射门力量。

## 射门力量建议

把拖动距离分成 3 档：

- 低力量：拖动距离短。
- 中力量：拖动距离适中。
- 高力量：拖动距离长。

力量效果：

- 低力量：命中率高，但球速慢，容易被扑。
- 中力量：最稳定，推荐作为默认手感。
- 高力量：球速快，更难扑，但有射偏或打门柱风险。

建议数值：

- 低力量：射偏概率 0%，被扑范围正常。
- 中力量：射偏概率 5%，被扑范围正常。
- 高力量：射偏概率 15%，守门员只有完全命中或提前移动相邻区域才容易扑到。

如果不想做随机，可以改成确定性规则：

- 高力量且拖拽角度太偏，判定为射偏或打门柱。
- 中力量正常命中。
- 低力量正常命中，但守门员相邻区域也可能扑到。

## 触球点建议

点击足球不同位置影响球路：

- 点击足球下部：球更容易飞向上半区。
- 点击足球上部：球更容易飞向下半区。
- 点击足球左侧：球路偏向右侧或带右弧线。
- 点击足球右侧：球路偏向左侧或带左弧线。
- 点击足球中心：球路更直，偏向中路或拖拽方向。

为了降低实现难度，可以只做简单映射：

- 点击上半球：目标高度偏低。
- 点击下半球：目标高度偏高。
- 点击左半球：目标方向偏右。
- 点击右半球：目标方向偏左。

## 守门员输入判定

守门员玩家操作：

1. 点击守门员。
2. 按住并向目标方向拖动。
3. 松开后守门员扑向该方向。
4. 松开后双击屏幕，触发提前移动。

守门员拖拽方向映射：

- 向左上拖：扑左上。
- 向左下拖：扑左下。
- 向右上拖：扑右上。
- 向右下拖：扑右下。
- 拖动较小或没有明显方向：保持中路。

## 提前移动规则

守门员松开后双击屏幕，触发提前移动。

提前移动效果：

- 扑救范围更大。
- 如果方向正确，更容易扑出。
- 如果方向相邻，也有机会碰到球。
- 如果方向错误，无法回中路，基本被进球。

建议规则：

- 未提前移动：只能扑到完全相同区域。
- 提前移动：可以覆盖目标区域和 1 个相邻区域。
- 高力量射门：提前移动只能覆盖相邻区域的一部分，可以表现为碰到但不一定扑出。

## 无输入中路防守

如果守门员玩家不输入：

- 足球踢向中上：守门员接住。
- 足球踢向中下：守门员用脚挡出。
- 足球踢向左上、左下、右上、右下：守门员无法扑到。

这个规则让“不动”成为一种明确策略：赌中路。

## 基础结果表

| 射门区域 | 守门员行为 | 结果 |
| --- | --- | --- |
| 与扑救区域相同 | 正常扑救 | 扑出 |
| 与扑救区域相同 | 提前移动 | 扑出，表现更强 |
| 与扑救区域相邻 | 正常扑救 | 进球 |
| 与扑救区域相邻 | 提前移动 | 根据力量判定 |
| 与扑救区域相反 | 任意扑救 | 进球 |
| 中路 | 守门员无输入 | 接住或脚挡 |
| 左右死角 | 守门员无输入 | 进球 |

## 相邻区域建议

相邻关系：

- 左上相邻：左下、中路。
- 左下相邻：左上、中路。
- 中路相邻：左上、左下、右上、右下。
- 右上相邻：右下、中路。
- 右下相邻：右上、中路。

简化规则：

- 只有提前移动才能覆盖相邻区域。
- 中路作为特殊区域，守门员无输入可以处理。

## 射偏和门柱规则

为了让高力量有风险，可以加入射偏和门柱。

建议简化：

- 低力量：不会射偏。
- 中力量：极少射偏。
- 高力量：如果拖拽角度接近球门边缘，可能打门柱或射偏。

结果优先级：

1. 如果判定射偏，直接射失。
2. 如果判定打门柱，播放门柱结果。
3. 如果没有射偏或门柱，再进行守门员扑救判定。

GameJam MVP 可以先不做门柱，只保留射偏。

## Slowmo 输入时间

总时长：5 秒。

建议节奏：

- 5 秒：进入 slowmo，显示球刚离脚。
- 4 秒：允许双方开始输入。
- 3 秒：输入最清晰，给出拖拽反馈。
- 2 秒：提示即将锁定。
- 1 秒：最后确认。
- 0 秒：锁定输入，恢复正常速度并播放结果。

超时规则：

- 点球玩家没有输入：默认低力量踢向中路。
- 守门员玩家没有输入：默认守中路。

## 结算流程

每次点球结算顺序：

1. 获取点球玩家输入。
2. 计算足球目标区域。
3. 计算射门力量。
4. 判断是否射偏或打门柱。
5. 获取守门员输入。
6. 判断守门员是否提前移动。
7. 判断扑救区域是否覆盖射门区域。
8. 播放对应动画。
9. 更新比分。
10. 交换攻守。

## 推荐 MVP 判定版本

如果时间紧，只做以下规则：

- 球门 5 区域。
- 点球玩家拖拽决定射门区域和力量。
- 守门员拖拽决定扑救区域。
- 守门员双击触发提前移动。
- 相同区域：扑出。
- 不同区域：进球。
- 提前移动：可扑相邻区域。
- 高力量：有小概率射偏。
- 守门员不输入：自动防中路。

## 调试建议

- 在开发阶段显示球门 5 区域边框。
- 显示当前射门区域文字。
- 显示当前扑救区域文字。
- 显示力量档位。
- 显示是否触发提前移动。
- 显示最终判定原因，例如“进球：守门员扑错方向”。

正式版本可以隐藏调试文字，只保留自然动画和 UI 反馈。
