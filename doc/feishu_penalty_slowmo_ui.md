# 点球 Slowmo：界面流程与 UI 文案

## 目标

这份文档用于指导《点球 Slowmo：54321》的界面流程、比赛 HUD 和关键 UI 文案。目标是让玩家不需要教程也能理解：怎么进房间、当前是谁射门、怎么操作、这一球为什么进或被扑。

## UI 设计原则

- 所有界面都优先服务快速试玩。
- 文案短，不讲长教程。
- 按钮大，鼠标容易点击。
- 比赛中只显示必要信息，避免挡住足球轨迹。
- `54321` 倒计时要足够醒目，是视觉中心之一。
- 进球、扑救、射失结果必须清楚有冲击力。

## 页面流程

推荐页面顺序：

1. 标题页
2. 模式选择页
3. 创建房间 / 加入房间页
4. 房间等待页
5. 双方准备页
6. 比赛页
7. 单球结算页
8. 比赛结算页

## 标题页

页面内容：

- 游戏标题：点球 Slowmo：54321
- 副标题：5 秒，决定胜负
- 主按钮：开始游戏
- 次按钮：操作说明

视觉建议：

- 背景使用球门和点球点。
- 标题上方或后方出现半透明巨大数字 `5 4 3 2 1`。
- 足球可以停在画面下方点球点。

## 模式选择页

按钮：

- 创建房间
- 加入房间
- 返回

GameJam 版本只做好友房即可。随机匹配可以先不做。

## 创建房间页

页面内容：

- 标题：创建房间
- 显示房间码
- 按钮：复制房间码
- 状态：等待对手加入
- 按钮：返回

文案建议：

- 房间码：`ABCD`
- 等待对手加入...
- 把房间码发给朋友

## 加入房间页

页面内容：

- 标题：加入房间
- 输入框：输入房间码
- 按钮：加入
- 按钮：返回

错误提示：

- 房间不存在
- 房间已满
- 网络连接失败
- 房间码不能为空

## 房间等待页

页面内容：

- 玩家 1：已加入
- 玩家 2：等待中 / 已加入
- 按钮：准备
- 状态：等待对手准备

文案建议：

- 你已准备
- 对手已准备
- 双方准备完成，比赛开始

## 比赛页 HUD

比赛页是核心界面，信息要少而清楚。

必须显示：

- 当前比分
- 当前轮次
- 5 轮点球记录
- 当前身份：你是点球手 / 你是守门员
- 当前阶段：准备主罚 / Slowmo / 结算中
- 倒计时数字

推荐布局：

- 顶部中央：比分，例如 `2 : 1`
- 顶部两侧：玩家名或玩家 A / 玩家 B
- 顶部下方：点球记录，例如 `○ × ○ - -`
- 屏幕中央上方：巨大倒计时数字
- 屏幕下方：操作提示和拖拽反馈

## 点球手操作 UI

显示时机：玩家是点球手，进入 slowmo 后。

UI 元素：

- 足球可点击高亮。
- 鼠标按住后显示拖拽箭头。
- 箭头长度表示力量。
- 箭头方向表示射门方向。
- 可显示简短提示：拖拽足球射门。

文案：

- 拖拽足球射门
- 松开确认
- 力量：低 / 中 / 高

注意：

- 不要用太多说明文字。
- 箭头和力量条比文字更重要。

## 守门员操作 UI

显示时机：玩家是守门员，进入 slowmo 后。

UI 元素：

- 守门员可点击高亮。
- 鼠标按住后显示扑救方向箭头。
- 松开后显示扑救目标。
- 双击后显示提前移动反馈。

文案：

- 拖拽守门员扑救
- 双击提前移动
- 守中路
- 提前移动！

注意：

- “双击提前移动”需要在第一次试玩时显示。
- 正式比赛中可缩短成图标或小提示。

## Slowmo 倒计时 UI

倒计时数字：

- 5：白色或亮色，进入 slowmo。
- 4：略微放大。
- 3：开始抖动。
- 2：颜色变暖或变红。
- 1：强烈冲击感。

视觉效果：

- 数字从画面中心或球门上方出现。
- 每次变化有短暂缩放。
- 最后一秒可以配合屏幕震动。
- 倒计时结束时数字破碎或快速淡出。

音效配合：

- 每秒一个清晰滴答声。
- 3 秒后心跳增强。
- 1 秒时音效突然收紧。

## 单球结算 UI

每球结束后显示 1-2 秒结果。

结果文案：

- 进球！
- 扑出！
- 接住了！
- 脚挡！
- 射偏！
- 门柱！
- 横梁！

补充说明文案：

- 守门员扑错方向
- 守门员守住中路
- 力量太大
- 提前移动成功
- 死角破门

注意：

- 主要结果用大字。
- 原因说明用小字。
- 不要停留太久，保持比赛节奏。

## 比赛结算页

页面内容：

- 大标题：你赢了 / 你输了
- 最终比分
- 点球记录
- 按钮：再来一局
- 按钮：返回房间
- 按钮：返回主页

胜利表现：

- 角色庆祝。
- 观众欢呼。
- 背景灯光加强。

失败表现：

- 角色懊恼。
- 观众叹息。
- 背景略暗。

## 点球记录显示

建议用简单符号：

- `○` 表示进球。
- `×` 表示未进。
- `-` 表示未踢。

示例：

```text
玩家 A：○ ○ × - -
玩家 B：○ × ○ - -
```

突然死亡时：

- 显示“突然死亡”
- 点球记录可以继续向后追加。

## 新手提示

只在第一次或等待阶段显示。

点球手提示：

- 点击足球，向后拖拽，松开射门。
- 拖得越远，力量越大。

守门员提示：

- 点击守门员，拖向你要扑救的方向。
- 不操作会守中路。
- 松开后双击可以提前移动。

## GameJam MVP UI

如果时间不够，只做以下 UI：

- 标题页。
- 创建房间。
- 加入房间。
- 房间等待。
- 比赛 HUD。
- slowmo 倒计时。
- 拖拽箭头。
- 比分板。
- 单球结果文字。
- 最终胜负页。

可以暂时不做：

- 操作说明页。
- 复杂动画 UI。
- 详细点球记录。
- 漂亮的胜负插画。
- 排行榜。

## 给 Agent 的注意事项

- 比赛 HUD 不要挡住球门和足球轨迹。
- 倒计时必须醒目。
- 操作提示要短，最好能用箭头表达。
- 结果文字要大，玩家一眼知道发生了什么。
- 联机等待状态一定要有提示，否则玩家会以为卡住了。
